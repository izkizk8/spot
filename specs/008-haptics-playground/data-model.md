# Phase 1 — Data Model: Haptics Playground

All entities below live in `src/modules/haptics-playground/types.ts` (or
co-located with the consumer module) and are pure TypeScript types — no
classes, no runtime schema library.

---

## `HapticKind`

The top-level dispatch axis the driver uses.

```ts
export type HapticKind = 'notification' | 'impact' | 'selection';
```

| Value | Underlying `expo-haptics` call |
|---|---|
| `'notification'` | `notificationAsync(NotificationFeedbackType.<intensity>)` |
| `'impact'` | `impactAsync(ImpactFeedbackStyle.<intensity>)` |
| `'selection'` | `selectionAsync()` (intensity ignored / must be undefined) |

---

## `NotificationIntensity` / `ImpactIntensity` / `HapticIntensity`

Mirror the `expo-haptics` enums but in the project's own string-literal
union form so tests don't have to import the third-party enum.

```ts
export type NotificationIntensity = 'success' | 'warning' | 'error';

export type ImpactIntensity = 'light' | 'medium' | 'heavy' | 'soft' | 'rigid';

export type HapticIntensity = NotificationIntensity | ImpactIntensity;
```

The driver maps these to the `expo-haptics` enum at the call site. The
mapping table is the authoritative contract — see
`contracts/haptic-driver.md`.

**Validation**:

- `play('notification', 'light')` is a *type error* (impact intensity used
  on a notification kind). Enforced by overload signatures on
  `driver.play` (see contract).
- `play('selection', anything)` is a *type error* — the second parameter
  must be omitted.

---

## `Cell`

A single composer cell. Either off, or a tagged haptic option.

```ts
export type Cell =
  | { readonly kind: 'off' }
  | { readonly kind: 'impact'; readonly intensity: ImpactIntensity }
  | { readonly kind: 'notification'; readonly intensity: NotificationIntensity };
```

**Cycle order** (FR-010), going from a starting `{kind:'off'}` and back:

```text
off → impact:light → impact:medium → impact:heavy
    → impact:soft  → impact:rigid
    → notification:success → notification:warning → notification:error
    → off
```

Selection is intentionally omitted from the cycle: it has no intensity, no
visual rank, and the spec lists exactly nine cycle states starting at off
(`off → 8 options → off`), matching the eight cells.

**Invariants**:

- `length(pattern) === 8` for every composer state and every persisted
  pattern.
- A cell with `kind === 'off'` has no other fields. Discriminated union
  enforces this at compile time.

---

## `Pattern`

```ts
export type Pattern = readonly [Cell, Cell, Cell, Cell, Cell, Cell, Cell, Cell];
```

A length-8 readonly tuple. Using a tuple (not `Cell[]`) makes
`pattern.length === 8` a compile-time guarantee.

---

## `Preset`

```ts
export interface Preset {
  /** Stable unique id, opaque string. Generated by presets-store.save(). */
  readonly id: string;

  /** Auto-generated human-readable name, e.g. "Preset 3". */
  readonly name: string;

  /** The 8-cell pattern. */
  readonly pattern: Pattern;

  /** ISO-8601 timestamp the preset was saved. */
  readonly createdAt: string;
}
```

**Validation rules** (applied in `presets-store.list()` per FR-025):

- `typeof id === 'string' && id.length > 0`
- `typeof name === 'string' && name.length > 0`
- `Array.isArray(pattern) && pattern.length === 8 && every(isValidCell)`
- `typeof createdAt === 'string'` (no parse — tolerated as opaque)

Entries failing any of the above are *skipped*. Valid entries in the same
payload are still returned.

## `PresetsStoreError`

```ts
export type PresetsStoreErrorCode = 'write-failed' | 'empty-pattern';

export class PresetsStoreError extends Error {
  readonly code: PresetsStoreErrorCode;
  constructor(code: PresetsStoreErrorCode, message: string) {
    super(message);
    this.code = code;
    this.name = 'PresetsStoreError';
  }
}
```

- `'write-failed'` — `AsyncStorage.setItem` threw or rejected.
- `'empty-pattern'` — caller passed an all-`off` pattern (FR-020); enforced
  in `save()` so the screen UI doesn't have to repeat the check.

`list()` and `delete()` *never* throw — they swallow errors and return safe
defaults (`[]` / `void`).

## State transitions

### Sequencer playback state machine

```text
idle ──Play (non-empty)──▶ playing ──last cell fires──▶ idle
   │                          │
   │                          └── Play / Save / unmount ──▶ idle (cancelled)
   │
   └── Play (empty) ──▶ idle (no-op, FR-015)
```

Implemented as a single boolean (`isPlaying`) plus a cancel-token ref. The
cancel token is replaced (not mutated) on every Play press; the chained
`setTimeout` callback compares its captured token to the current ref before
firing the next cell, and short-circuits if they differ.

### Presets list state

```text
mount ──list()──▶ loaded(presets) ──save()──▶ loaded(presets ∪ new)
                                  └─delete──▶ loaded(presets ∖ deleted)

mount ──list() throws──▶ loaded([]) (silent, FR-024)
```

The screen treats `loaded([])` and "load failed" identically by design — no
error UI is shown for a read failure (it's indistinguishable from a fresh
install, which is also `[]`).

## Storage layout

**Key**: `spot.haptics.presets` (FR-018).
**Value**: `JSON.stringify(Preset[])`.

Example payload:

```json
[
  {
    "id": "1714238400000-7k3p2x",
    "name": "Preset 1",
    "pattern": [
      { "kind": "impact", "intensity": "light" },
      { "kind": "impact", "intensity": "light" },
      { "kind": "impact", "intensity": "light" },
      { "kind": "off" },
      { "kind": "notification", "intensity": "success" },
      { "kind": "off" },
      { "kind": "off" },
      { "kind": "off" }
    ],
    "createdAt": "2026-04-27T12:00:00.000Z"
  }
]
```

## Summary

Bring 5 Expo and React Native packages up to the versions Expo recommends for SDK 55 so `pnpm start` no longer prints the outdated-dependencies warning. Mechanical upgrade only — no source-code changes anticipated, no major version bump.

**Scale/Scope**: 5 package version bumps; `package.json` + `pnpm-lock.yaml` only.

## Approach

1. Run `npx expo install --fix` in the worktree. This rewrites the 5 entries in `package.json` to their recommended ranges and reruns `pnpm install` to regenerate `pnpm-lock.yaml`.
2. Verify with `npx expo install --check` that no packages remain outdated.
3. Run the local quality gate: `pnpm typecheck`, `pnpm test`, `pnpm lint`, `pnpm docs:check`.
4. Smoke-test `pnpm start` to confirm the warning is gone (kill the dev server after Metro boots).
5. Normalise CRLF line endings on `package.json` and the new spec artifacts before commit (per `pnpm docs:check` policy).
6. Single commit + PR with the `Co-authored-by: Copilot` trailer.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `expo-location` 19→55 introduces a hidden behaviour change | CHANGELOG audit shows no breaking API changes; `pnpm typecheck` + jest tests catch type regressions; manual smoke later if shipped to a build. |
| `react-native-maps` 1.15→1.27 changes a prop signature used by `mapkit-lab` | TypeScript catches removed/renamed props at typecheck; release notes list only additive changes. |
| `pnpm-lock.yaml` resolves a transitive dep to a version EAS doesn't like | `npx expo install --check` is the canonical compatibility check; we rerun it post-install to confirm clean state. |
| EAS auto-build triggered on push consumes minutes prematurely | Branch is not tagged; EAS GitHub integration is tag/branch-filtered. No release tag is pushed in this feature. |

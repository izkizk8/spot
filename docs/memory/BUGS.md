# Bug Patterns

No bug patterns recorded yet.

Add an entry here when a failure mode is:
- **Repeatable** — likely to recur in future features
- **Non-obvious** — the root cause was surprising or counterintuitive
- **Evidenced** — backed by a failing test, production incident, or verified fix
- **Preventable** — there is a concrete action that future work should take to avoid it

Use the format below for each entry:

### YYYY-MM-DD - Bug / Failure Pattern
**Status**: Active | Monitored | Retired
**Symptoms**: What was observed?
**Root Cause**: What actually caused it?
**Future mistake prevented**: What change pattern should future work avoid?
**Evidence**: Failing test, production incident, review finding, or verified fix.
**Prevention / Detection**: How should future work avoid it and how can we catch it sooner?
**Where to look next**: Files, modules, logs, or checks maintainers should inspect.

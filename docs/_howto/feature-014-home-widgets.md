# How to verify Home Screen Widgets on iPhone

## Goal
Confirm the Spot Showcase widget appears in the iOS widget gallery at Small,
Medium, and Large sizes, renders the default configuration, and refreshes when
the Widgets Lab screen pushes a new configuration.

## Prerequisites
- macOS with Xcode 15+ (or EAS Build cloud)
- iPhone running iOS 14+
- `with-home-widgets` config plugin registered in `app.json`
- `pnpm check` green; dev client built with widget extension

## Steps
1. Build the JS layer:
   ```bash
   pnpm install && pnpm check
   ```
2. Prebuild and build the dev client IPA:
   ```bash
   npx expo prebuild --clean
   eas build --profile development --platform ios --local
   ```
3. Install the IPA per [sideload-iphone.md](sideload-iphone.md).
4. Open the app and navigate to **"Widgets Lab"** in the Modules tab.
5. Confirm the status panel, configuration panel, setup-instructions card,
   and three preview cards are visible.
6. Long-press the Home Screen → tap **+** → search **"spot showcase"**.
7. Confirm exactly one entry titled "spot showcase" with exactly three sizes.
8. Add the Small widget to the Home Screen.
9. Back in the app, set showcase value, bump counter to 42, tap green tint.
10. Tap **Push to widget** — confirm within ~1 s all widgets refresh with the new values.

## Verify
- Widget appears in gallery with exactly three sizes (Small, Medium, Large)
- Default config renders on first add (before app interaction)
- Push to widget refreshes all installed sizes within ~1 second
- Reload event log shows entries (capped at 10); resets on screen unmount
- On Android/web: "Home Screen Widgets are iOS only" banner, Push button disabled

## Troubleshooting
- **Widget not appearing in gallery** → confirm `with-home-widgets` plugin is in
  `app.json` and `SpotWidgetBundle.swift` was generated by prebuild
- **Widget shows "Hello, Widget!" after push** → verify App Group entitlement
  `group.com.izkizk8.spot.showcase` is on both app and extension targets
- **Broken widget chrome** → run `npx expo prebuild --clean` and rebuild
